## 2024-05-30 - [Performance] Removing always_run = timestamp() to preserve idempotency and speed up applies
**Learning:** Using `always_run = timestamp()` on initialization or download provisioners like `null_resource.download_bank_of_anthos` forces the resource to be replaced on every `terraform apply`. Because downstream application deployments (like `null_resource.deploy_bank_of_anthos`) trigger off the `download_id`, this causes a cascading replacement chain that unnecessarily re-deploys the entire application and slows down the apply process significantly.
**Action:** When implementing provisioners, rely on functional triggers (like `version`) to determine when an update is needed. Avoid using `always_run = timestamp()` unless strictly necessary for the intended behavior, and break the downstream replacement chain by removing dynamic upstream triggers (like `download_id`) from application deployment resources. Ensure idempotency is maintained in the bash script itself (e.g. `curl -L -o ...`).
