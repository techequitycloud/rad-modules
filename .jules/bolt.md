## 2024-05-02 - Eliminate redundant downloads in local-exec null_resources
**Learning:** Using `always_run = timestamp()` or a dynamic trigger like `manifests_missing = fileexists(...) ? "exists" : timestamp()` in `null_resource` to execute initialization jobs has negative consequences. `always_run` executes on every apply, slowing down deployments. Using `fileexists(...)` causes an unintended two-apply state issue, forcing resource recreation on the second apply because the trigger value changes from a timestamp to "exists".
**Action:** When a `null_resource` downloads or generates local files, remove dynamic time-based triggers. Instead, rely on static triggers (like version numbers) and add bash logic within the `local-exec` provisioner to skip downloading if the files already exist.
