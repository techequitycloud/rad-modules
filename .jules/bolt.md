## 2024-06-01 - Avoid timestamp triggers for idempotency
**Learning:** Setting `always_run = timestamp()` on a `null_resource` ensures execution triggers resource replacement on every apply. This defeats any bash idempotency (like `if [ -d "$dir" ]; then exit 0; fi`) because the `when = destroy` cleanup provisioner runs before the new create provisioner, deleting files. Downstream resources depending on the `download_id` will also be unnecessarily replaced.
**Action:** To speed up applies and prevent downtime, remove the `always_run` trigger entirely so the provisioner only runs when specific functional triggers (like `version`) change. Break the downstream replacement chain by removing dynamic upstream triggers (like `download_id`), relying only on `depends_on` for sequencing.
