## 2024-06-02 - Eliminate downstream replacement chains from null_resources
**Learning:** Using `always_run = timestamp()` on a `null_resource` trigger ensures the resource replacement is forced on every apply. If the resource has a `when = destroy` provisioner, the resource will be destroyed (e.g. deleting files) before being recreated, nullifying the idempotency of the shell script. Furthermore, linking downstream resources through dynamic outputs like `download_id` forces those dependent resources to be replaced on every apply as well. This severely degrades `terraform apply` performance and risks unwanted downtime for dependent application resources.
**Action:** Remove `always_run = timestamp()` from triggers when possible to only execute when functional triggers change. To break downstream replacement chains, rely entirely on `depends_on = [...]` instead of dynamic upstream attributes.
