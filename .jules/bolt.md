## 2024-06-09 - Remove always_run timestamp in null_resource
**Learning:** Setting `always_run = timestamp()` on a `null_resource` forces resource replacement on every apply. If the resource contains a `when = destroy` cleanup provisioner, this forced replacement will delete files before the new create provisioner runs, defeating any bash idempotency (like `if [ -d "$dir" ]; then exit 0; fi`). Furthermore, dynamic triggers (like `download_id = null_resource.download.id`) propagate this replacement to downstream resources.
**Action:** To speed up applies and prevent downtime, remove the `always_run` trigger entirely so the provisioner only runs when specific functional triggers (like `version`) change. Break the downstream replacement chain by removing dynamic upstream triggers (like `download_id`), relying only on `depends_on` for sequencing.
