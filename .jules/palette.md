## 2024-05-12 - Missing Standard Output Description Consistency\n**Learning:** The modules have an inconsistent description for the `external_ip` output or are missing it entirely in some modules, while GKE/mesh modules should provide a standard way to retrieve the gateway or service IP.\n**Action:** Add clear, consistent descriptions to outputs in `outputs.tf` across modules.
## 2024-05-12 - Variables Misplaced in group=0
**Learning:** Variables that belong to specific sections (like Network `group=2`, GKE `group=3`, Features `group=4`, Application `group=6`) are sometimes mistakenly placed in `group=0` (the Main/Provider group) while having order numbers that match their intended groups (e.g., `group=0 order=401` for `enable_services` in Istio_GKE). This causes them to appear on the wrong wizard page, confusing deployers.
**Action:** When auditing UIMeta, check that the `group=N` value matches the logical section the variable belongs to, especially when the `order` number is `> 200`.
