## 2024-05-17 - Missing Standard Outputs in GKE Modules
**Learning:** Discovered that some GKE application modules (like Bank_GKE and MC_Bank_GKE) omit the standard `cluster_credentials_cmd` and `external_ip` outputs mandated by `SKILLS.md`. This forces operators to manually reconstruct `gcloud container clusters get-credentials` commands and manually search the GCP console for the application's global load balancer IP, severely degrading operator experience during post-deployment verification.
**Action:** When auditing module outputs, ensure `cluster_credentials_cmd` is always present for GKE modules. For modules provisioning a global load balancer, `external_ip` should directly output the load balancer's address resource (e.g., `google_compute_global_address.glb.address`) instead of relying on a local file, ensuring reliable output generation.
