## 2024-05-28 - Over-permissioned Default Node Service Accounts in GKE Modules\n**Vulnerability:** The default GKE node service accounts in `Bank_GKE`, `Istio_GKE`, and `MC_Bank_GKE` modules were granted `roles/storage.objectAdmin`, which provides project-wide destructive write access to all GCS buckets.\n**Learning:** Default node service accounts are shared across all pods running on the nodes unless Workload Identity is configured. Granting broad write access (like `objectAdmin`) to the node service account violates least privilege and allows any compromised pod (even those without Workload Identity) to potentially delete or overwrite data in any bucket within the project. The nodes typically only need `roles/storage.objectViewer` to pull container images from GCR/Artifact Registry.\n**Prevention:** Never grant `roles/storage.objectAdmin` or other broad write roles to GKE node service accounts. Rely strictly on `roles/storage.objectViewer` for image pulls. If specific applications require write access to GCS, use Workload Identity with dedicated, narrowly-scoped service accounts.
